Key Concepts

While Trio is rather an easy application to learn and use, to truly take advantage of all its features does require that you understand the following key concepts.

Project Assets

Creating web pages and web sites with Trio involves the use of assets. The following is a list of all the various assets which are available to you when building your website with Trio:

  • templates: always .html files which contains markup and do not include YAML front matter.
  • fragments: either .html or .md files which contain either markup or markdown content and which must always include YAML front matter at the very top of the file.
  • includes: either .html or .md files which contain either markup or markdown content and which may or may not include YAML front matter at the very top of the file.
  • JSON data: JSON data that you provide to your project in .json data files.
  • metadata: all the data that Trio collects from your project's template, fragment, include and JSON data assets and which Trio makes available to your project's tag-based JavaScript callbacks to use.
  • tag-based JavaScript callbacks: JavaScript Node modules that export a single function which Trio calls when composing your web pages.

Page Composition

Basic HTML Page Composition

At a minimum, every webpage that you want to create is composed of a single template asset and a single fragment asset: HTML Page = Template Asset + Fragment Asset.

  • In the template that the fragment is associated with you pick a single HTML tag that you want the fragment's content to be appended to or to replace.
  • To mark the HTML tag in the template that you picked as a target for the fragment's content you add a data-trio-fragment attribute.
  • In the fragment you add YAML front matter to the very top of the file with a "template" property whose value names the template that the fragment is associated with.

The HTML page that Trio creates will be composed of the content contributed by the template asset and the content contributed by the fragment asset added to the tag in the template that is marked as a target using the data-trio-fragment attribute.

Basic HTML page composition allows different fragment assets to be associated with the same template asset, where the template asset provides the basic structure and content of a web page and where each fragment asset tailors the page with its own unique structure and content.

Advanced HTML Page Composition

While basic HTML page composition as described above might suffice to create very simple web pages that don't vary much in the way of design, you will often find that your web pages, though similar, require a greater degree of refinement than what basic HTML page composition provides.

Includes

Using include assets allow you additional control over how you compose your web pages. Their use-case is similar in nature to reusing common code by wrapping the code in functions, such as when coding in a programming language like JavaScript.

For example, many web sites share the same footer markup on most if not all of their web pages and using include assets to add common blocks of markup to your web pages makes implementing common structure and content like that across numerous web pages simple.

  • Include assets can be declared in both template assets and fragment assets and you aren't limited to just one as fragments are.
  • To include an include asset into a template asset or into a fragment asset you pick an HTML tag in the asset that you want to either append to or replace with the include asset's content.
  • To mark the HTML tag that you picked in the template or fragment as a target for an include asset's content you add a data-trio-include attribute and assign the name of the include file (or a reference to a YAML front matter property to be found in the fragment asset whose value is the name of the file) as the attributes value.

HTML Tag-Based JavaScript Callbacks

Unlike other static website generators, Trio doesn't use a templating engine such as Handlebars or Liquid Templates, for example. Instead, Trio exposes its underpinnings, which is JavaScript, to you through tag-based JavaScript callbacks, allowing you to use JavaScript to dynamically compose your web pages with it, which is kind of awesome if you think about it.

And because Trio is built on top of Node, you have all the latest ES6+ goodies that it implements at your disposal without having to transpile. In addition, and this is a biggie, your markup stays clean and readable because it isn't littered with scattered bits of templating statements.

  • Internally Trio treats all you markup and markdown as simple text and parses it using jQuery-like selector semantics using an awesome OS library called Cheerio, which you should acquaint yourself with but if you have ever used jQuery then you should already know how to use Cheerio's selector semantics.
  • Tag-based JavaScript callbacks can be assigned to HTML tags in your templates, fragments and includes simply by adding data-trio-callback attributes to them and assigning them the names of the JavaScript modules that you want to be called as their values.
  • After creating your web page's composite (the result of its template, fragment and any include assets), Trio scans the composite for any tag-based callbacks and calls each one asynchronously, passing each one an object whose properties can be destructured and used with Cheerio to manipulate the composite as if you were actually using jQuery to manipulate the DOM.

HTML Tag Attributes

As you may have noticed while reading this page, numerous HTML tag attributes were discussed, such as the data-trio-fragment and data-trio-include. These and other data-trio-* attributes are markers and their presence indicates to Trio that it is to take certain actions when it encounters one of them, such as appending or replacing the tag with the content from a fragment or an include.

YAML

YAML, which stands for "Yet Another Markup Language", is used to assign properties to various assets in your project.

Trio predefines numerous front matter properties that it expects to find in your assets, which it uses internally when composing your web pages.

For example, Trio retrieves the name of the fragment's associated template from the fragment's predefined front matter template property.

In addition to Trio's predefined front matter properties, your can also define custom front matter properties in fragment assets.

All the front matter properties from all the composite's assets are accessible to all the composite's tag-based JavaScript callbacks.

Trio uses the awesome OS library called gray-matter to implement its YAML front matter support.

JSON Data

You can create any number of JSON data-files and use the data they contain in your tag-based JavaScript callbacks to compose your pages.

You can hand-code these data files or you can create NPM tasks that you run to either retrieve the data from a local or remote database or from a remote service.

Metadata

Trio collects metadata from your project's assets and configuration file and uses it internally to build your site. Trio also makes this data accessible to your project's tag-based JavaScript callbacks.

Trio organizes its metadata in a structured format that makes using it easier to reason about.

For example, you can use this metadata in your tag-based JavaScript callbacks to:

  • Create links to other web pages on your web site.
  • Generate content based on JSON data or from custom front matter properties.
  • Make lists of blog articles related by their categories and tags.
  • Construct links to the next and and previous blog article based on their published dates.

After each build of your project, Trio persists most of the metadata that it collects to a file named root/trio.manifest.json. You can view the contents of this file to assist you when you are coding your tag-based JavaScript callbacks.

Trio's use of common metaphors for its project assets, its approach to page composition, its organized collections of metadata and its tag-based JavaScript callback mechanism serve to create a powerful but easy to learn and use platform upon which to create your static sites.
