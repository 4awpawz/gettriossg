Front Matter

The inclusion of front matter is required for fragment assets and is optional for include and tag-based callback assets.

Front matter must be placed at the very top of the file and it must be surrounded by opening and closing delimiters that are appropriate for the asset's file type.

Excerpts are supported in fragment assets but are not supported in include and tag-based callback assets.

Appropriate Delimiters By Asset Type

Fragment and include assets use opening and closing HTML comment tags "<!--", "-->" as their front matter delimiters.

Tag-based callbacks use opening and closing JavaScript block comment tags "/*", "*/" as their front matter delimiters.

Predefined Front Matter Properties

Trio predefines numerous required and optional front matter properties for you to use in your fragment and include asset's front matter.

All Fragments

  • template: required, type string and is used to name the template that the fragment is associated with
  • title: required, type string and is used to set the title tag of the page
  • appendToTarget: optional, type boolean, default value false
    • if true Trio appends the fragment's content to the tag in the template that is decorated with the data-trio-fragment attribute
    • if false Trio replaces the tag in the template that is decorated with the data-trio-fragment attribute with the fragment's content
  • alwaysBuild: optional, type boolean, default value false
    • if true then Trio will always treat this fragment as if it is stale when building your project incrementally
    • this should be used when generating composites that consume the metadata generated from assets that are unrelated to this composite, such as when generating pages that contain lists of other pages (e.g. blog pages, tag pages, category pages, catalog pages, portfolio pages, etc.)
    • see Declaring External Dependencies To Trigger Incremental Builds for details
  • custom properties: supported, type any supported front matter data type

Blog Specific Fragments

In addition to the above, blog-specific fragment assets - tag, category and article - support the following additional front matter properties.

Tag Fragments

  • forTag: optional, type string, default value none and can be used in tag-based callbacks to filter blog articles by their tag property.

Category Fragments

  • forCategory: optional, type string, default value none and can be used in tag-based callbacks to filter articles by their category property.

Article Fragments

  • tag: optional, type string, default value none and can be used in tag-based callbacks to find all matching blog articles by their tag property.
  • category: optional, type string, default value none and can be used in tag-based callbacks to find all matching blog articles by their category property.

Includes

  • appendToTarget: optional, type boolean, default value false
    • if true Trio appends the include's content to the tag in the fragment or template that is decorated with the data-trio-include attribute that names the include
    • if false Trio replaces the tag in the fragment or template that is decorated with the data-trio-include attribute that names the include with the include's content
  • custom properties: not supported

Tag-Based Callbacks

  • dataDependencies: optional, type list, default value none
    • a list of JSON data file dependencies
    • see Declaring Internal Dependencies To Trigger Incremental Builds for details
  • moduleDependencies: optional, type list, default value none
    • a list of Node module dependencies.
    • see Declaring Internal Dependencies To Trigger Incremental Builds for details
  • custom properties: not supported

Blog Related Data Catalogs For Implementing Sophisticated Blog Sites

Trio uses the above blog-specific front matter properties to generate numerous blog related data catalogs for you. You can access these data catalogs in your tag-based callbacks and you can view them in file root/trio.manifest.json.

There are many ways you can take advantage of these data catalogs, such as using them to create lists of articles grouped by tags in your tag pages or lists of articles grouped by category in your category pages, for instance. These are common elements in many sophisticated blog sites and Trio makes it easy for you to implement them in your site.
