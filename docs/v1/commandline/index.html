Command Line

The following commands are available to you in your terminal application:

  • trio -h: display general help information
  • trio -h build | release | serve: display command specific help information
  • trio new path/to/project: creates an empty project in path/to/project
  • trio build | b: builds your site
  • trio build | b -i: incrementally builds your site
  • trio build | b -w: builds your site any time a change is made to a file in the root/source folder
  • trio build | b -iw: incrementally builds your site any time a change is made to a file in the root/source folder
  • trio build | b -ws: builds your site any time a change is made to a file in the root/source folder and serves it in the default browser
  • trio build | b -iws: incrementally builds your site any time a change is made to a file in the root/source folder and serves it in the default browser
  • trio release | r: build the site for release
  • trio release | r -b: build the site for release with cache busting applied
  • trio serve | s: serves your site in the default browser

Build Vs. Release

Building for development, or just building, is what you do while you are developing your site.

Building for release is what you do just prior to deploying your site to its server. When building for release, Trio doesn't generate a CSS map file, doesn't generate pages whose fragments were marked as a WIP, cleans your site's pages of all their data-trio-* tag attributes and provides the option to cache bust the generated site.
