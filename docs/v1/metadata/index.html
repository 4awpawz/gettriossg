Metadata

Trio organizes the metadata that it exposes to tag-based JavaScript callbacks in a structured format that makes using it easier to reason about.

After each build of your project, Trio persists most of the metadata that it collects to a file named root/trio.manifest.json. You can view the contents of this file to assist you when coding tag-based JavaScript callbacks.

Below is a list of all the metadata that is collected and made available to tag-based JavaScript callbacks:

  • timestamp: the date and time of the current build in "YYYY-MM-DD HH:MM:SS" format
  • version: the version of Trio used to build the project in semantic versioning format
  • buildType: either "development" or "release", depending on your build options
  • userConfig: the configuration options used to build the project taken from root/trio.json at time of build
  • frags: an array of all non blog-article fragments
  • wipsCount: total number of "work in progress" fragments
  • articlesCount: total number of blog articles
  • articlesCatalog: an array of blog articles, each with their related articles by tag, related articles by tag flattened and related articles by category lists, in ascending order by article date and article name
  • categoriesCatalog: an array of blog article categories, each with their related articles lists, in ascending order by category
  • sortedTagCatalog: an array of blog articles tags, each with their related articles list, in ascending order by tag
  • dataCatalog: an object with properties for each JSON file in root/source/data - please note that this catalog is not persisted to file root/trio.manifest.json
