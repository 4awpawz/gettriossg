Metadata

Trio organizes the metadata that it exposes to tag-based JavaScript callbacks in a structured format that makes using the data easier to reason about.

After each build of your project, Trio persists most of the metadata that it collects to a file named root/trio.manifest.json. You can view the contents of this file to assist you when coding tag-based JavaScript callbacks.

Trio calls each tag-based JavaScript callback with a single argument whose properties are $, $tag, asset and site, which provide access to its metadata.

Property $

The $ property is a Cheerio wrapper around the composite that contains the tag-based JavaScript callback tag.

Property $tag

The $tag property is a Cheerio wrapper around the tag whose data-trio-callback attribute initiated the callback.

Property asset

The asset property is an object containing all the metadata associated with a fragment.

asset.path

The path property is the path to the fragment file in the source/fragment folder. It is used internally by Trio during page generation.

asset.mtimeMs

The mtimeMs property is a timestamp that represent the time, in milliseconds, when the fragment was last modified. It is used internally by Trio to determine a file's staleness for incremental builds.

asset.type

The type property is a string which indicates the type of asset that a file is and is used internally by trio for incremental builds.

asset.isStale

The isStale property is a boolean which indicates if the fragment was marked as stale and is used internally by trio for incremental builds.

asset.matter

The matter property is the fragment's front matter.

asset.matter.content

The matter.content property is a string which contains the fragment's HTML content.

asset.matter.data

The matter.data property is an object with properties for each property declared in the front matter.

asset.matter.isEmpty

The matter.isEmpty property is a boolean which indicates if the front matter's content property is empty.

asset.matter.excerpt

The matter.excerpt property is a string which contains the HTML, if any, from the front matter's excerpt.

asset.id

The id property is an integer whose value is the unique ID. It is used internally by Trio. It can be used to filter for fragments from site.frags, site.articlesCatalog, site.categoriesCatalog, site.tagsCatalog, asset.relatedArticlesByCategory, asset.relatedArticlesByTag and assetRelatedArticlesByTagFlattened. Do not rely on this value to remain constant from one build to next. Always retrieve it from the front matter directly.

asset.destPath

The destPath property contains the path to the page in the the public folder.

asset.url

The url property contains the URL of the page.

asset.articleDate

Only relevant for blogging sites, the articleDate property is a string which contains the date of the article in "YYYY-MM-DD" format.

asset.nextArticleUrl

Only relevant for blogging sites, the nextArticleUrl property contains the URL to the next article.

asset.previousArticleUrl

Only relevant for blogging sites, the previousArticleUrl property contains the URL to the previous article.

asset.relatedArticlesByCategory

Only relevant for blogging sites that use tags, the relatedArticlesByCategory property is an object that catalogs other articles related to this article by its category.

asset.relatedArticlesByCategory.category

The relatedArticlesByCategory.category property is the page's category.

asset.relatedArticlesByCategory.related

The relatedArticlesByCategory.related property is an array of related articles in descending order by article date and article title.

asset.relatedArticlesByCategory.related[n].articleDate

The relatedArticlesByCategory.related[n].articleDate property is a string which contains the date of the article in "YYYY-MM-DD" format.

asset.relatedArticlesByCategory.related[n].url

The relatedArticlesByCategory.related[n].url property is the related article's URL.

asset.relatedArticlesByCategory.related[n].title

The relatedArticlesByCategory.related[n].title property is the related article's title.

asset.relatedArticlesByCategory.related[n].id

The id property is an integer whose value is the unique ID. It is used internally by Trio. It can be used to filter for fragments from site.frags, site.articlesCatalog, site.categoriesCatalog, site.tagsCatalog, asset.relatedArticlesByCategory, asset.relatedArticlesByTag and assetRelatedArticlesByTagFlattened. Do not rely on this value to remain constant from one build to next. Always retrieve it from the front matter directly.

asset.relatedArticlesByCategory.related[n].excerpt

The relatedArticlesByCategory.related[n].excerpt property is a string which contains the HTML, if any, from the front matter's excerpt.

asset.relatedArticlesByTag

Only relevant for blogging sites that use tags, the relatedArticlesByTag property is an array that catalogs other articles related to this article grouped in ascending order by tag.

asset.relatedArticlesByTag[n].tag

The relatedArticlesByTag[n].tag property is an article tag.

asset.relatedArticlesByTag[n].related

The relatedArticlesByTag[n].related property is an array of articles related to this article by asset.relatedArticlesByTag[n].tag.

asset.relatedArticlesByTag[n].related[n].articleDate

The relatedArticlesByTag[n].related[n].articleDate property is a string which contains the date of the article in "YYYY-MM-DD" format.

asset.relatedArticlesByTag[n].related[n].url

The relatedArticlesByTag[n].related[n].url property is the related article's URL.

asset.relatedArticlesByTag[n].related[n].title

The relatedArticlesByTag[n].related[n].title property is the related article's title.

asset.relatedArticlesByTag[n].related[n].id

The id property is an integer whose value is the unique ID. It is used internally by Trio. It can be used to filter for fragments from site.frags, site.articlesCatalog, site.categoriesCatalog, site.tagsCatalog, asset.relatedArticlesByCategory, asset.relatedArticlesByTag and assetRelatedArticlesByTagFlattened. Do not rely on this value to remain constant from one build to next. Always retrieve it from the front matter directly.

asset.relatedArticlesByTag[n].related[n].excerpt

The relatedArticlesByTag.related[n].excerpt property is a string which contains the HTML, if any, from the front matter's excerpt.

asset.relatedArticlesByTagFlattened

The relatedArticlesByTagFlattened property is an array that catalogs other articles related to this article and is similar to asset.relatedArticlesByTag but flattened for when you aren't concerned about each related article's tag.

asset.relatedArticlesByTagFlattened[n].related

The relatedArticlesByTagFlattened[n].related property is an array of relate articles.

asset.relatedArticlesByTagFlattened[n].related[n].articleDate

The relatedArticlesByTagFlattened[n].related[n].articleDate property is a string which contains the date of the article in "YYYY-MM-DD" format.

asset.relatedArticlesByTagFlattened[n].related[n].url

The relatedArticlesByTagFlattened[n].related[n].url property is the related article's URL.

asset.relatedArticlesByTagFlattened[n].related[n].title

The relatedArticlesByTagFlattened[n].related[n].title property is the related article's title.

asset.relatedArticlesByTagFlattened[n].related[n].id

The id property is an integer whose value is the unique ID. It is used internally by Trio. It can be used to filter for fragments from site.frags, site.articlesCatalog, site.categoriesCatalog, site.tagsCatalog, asset.relatedArticlesByCategory, asset.relatedArticlesByTag and assetRelatedArticlesByTagFlattened. asset.relatedArticlesByTagFlattened.related[n].excerpt

The relatedArticlesByTagFlattened.related[n].excerpt property is a string which contains the HTML, if any, from the front matter's excerpt.

Property site

The site property provides information about the site.

site.timestamp

The timestamp property is the date and time of the current build in "YYYY-MM-DD HH:MM:SS" format.

site.version

The version property is the version of Trio used to build the project in semantic versioning format.

site.buildType

The buildType property can be either "development" or "release", depending on the build options.

site.userConfig

The userConfig property contains the contents of the trio.json file.

site.frags

The frags property is an array of all non blog-article fragments. See Property asset above for information pertaining to each array item's format.

site.wipsCount

The wipsCount property is an integer whose value is the site's total number of "work in progress" fragments.

site.articlesCount

The articlesCount property is an integer whose value is the site's total number of blog articles.

site.articlesCatalog

Only relevant for blogging sites, the articlesCatalog property is an array that catalogs the site's blog article's sorted in ascending order by article date and article title. See Property asset above for information pertaining to each array item's format.

site.categoriesCatalog

Only relevant for blogging sites that use categories, the categoriesCatalog property is an array that catalogs the site's blog articles grouped in ascending order by category.

site.categoriesCatalog[n].category

The categoriesCatalog[n].category property is the category.

site.categoriesCatalog[n].related

The site.categoriesCatalog[n].related property is an array of related articles in descending order by article date and article title.

site.categoriesCatalog[n].related[n].articleDate

The categoriesCatalog[n].related[n].related[n].articleDate property is a string which contains the date of the article in "YYYY-MM-DD" format.

site.categoriesCatalog[n].related[n].url

The categoriesCatalog[n].related[n].url property is the related article's URL.

site.categoriesCatalog.related[n].title

The categoriesCatalog.related[n].title property is the related article's title.

site.categoriesCatalog.related[n].id

The id property is an integer whose value is the unique ID. It is used internally by Trio. It can be used to filter for fragments from site.frags, site.articlesCatalog, site.categoriesCatalog, site.tagsCatalog, asset.relatedArticlesByCategory, asset.relatedArticlesByTag and assetRelatedArticlesByTagFlattened. Do not rely on this value to remain constant from one build to next. Always retrieve it from the front matter directly.

site.categoriesCatalog.related[n].excerpt

The categoriesCatalog.related[n].excerpt property is a string which contains the HTML, if any, from the front matter's excerpt.

site.tagsCatalog

Only relevant for blogging sites that use tags, the tagsCatalog property is an array that catalogs the site's blog articles grouped in ascending order by tag.

site.tagsCatalog[n].tag

The tagsCatalog[n].tag property is the tag.

site.tagsCatalog[n].related

The site.tagsCatalog[n].related property is an array of related articles in descending order by article date and article title.

site.tagsCatalog[n].related[n].articleDate

The tagsCatalog[n].related[n].related[n].articleDate property is a string which contains the date of the article in "YYYY-MM-DD" format.

site.tagsCatalog[n].related[n].url

The tagsCatalog[n].related[n].url property is the related article's URL.

site.tagsCatalog.related[n].title

The tagsCatalog.related[n].title property is the related article's title.

site.tagsCatalog.related[n].id

The id property is an integer whose value is the unique ID. It is used internally by Trio. It can be used to filter for fragments from site.frags, site.articlesCatalog, site.categoriesCatalog, site.tagsCatalog, asset.relatedArticlesByCategory, asset.relatedArticlesByTag and assetRelatedArticlesByTagFlattened. Do not rely on this value to remain constant from one build to next. Always retrieve it from the front matter directly.

site.tagsCatalog.related[n].excerpt

The tagsCatalog.related[n].excerpt property is a string which contains the HTML, if any, from the front matter's excerpt.

site.dataCatalog

The dataCatalog property is an array that catalogs the site's JSON files in root/source/data along with their content. Each item in the array represents a single JSON file. Please note that this catalog is not persisted to the root/trio.manifest.json file. See JSON Data for more information.
