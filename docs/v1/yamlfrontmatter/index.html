YAML Front Matter

YAML front matter must be placed at the very top of a fragment and include file and must be delimited by opening and closing HTML comment tags, between which you add both predefined and custom front matter properties.

Predefined Front Matter Properties

Fragments

The inclusion of front matter in fragments is required.

  • template: required, type string
    • is used to name the template that the fragment is associated with
    • when creating the composite, Trio will merge the fragment's content with the target tag in the template that contains the data-trio-fragment attribute
  • title: required, type string and is used to set the title tag of the web page
  • appendToTarget: optional, type boolean, default value false
    • if set to true Trio appends the fragment's content to the template's target tag
    • if set to false Trio replaces the the template's target tag with the fragment's content
  • custom properties: supported, type any supported front matter data type

Includes

The inclusion of front matter in includes is optional.

  • appendToTarget: optional, type boolean, default value false
    • if set to true Trio appends the include's content to the template's or fragment's target tag
    • if set to false Trio replaces the the template's or fragment's target tag with the include's content
  • custom properties: not supported

Predefined Blog Specific Front Matter Properties

Tag, category and article fragment assets support the following additional blog-specific front matter properties.

Tag Fragments

  • forTag: optional, type string, default value none
    • can be used in tag-based JavaScript callbacks to find all matching blog articles by their tag property value.

Category Fragments

  • forCategory: optional, type string, default value none
    • can be used in tag-based JavaScript callbacks to find all matching blog articles by their category property value.

Article Fragments

  • tag: optional, type string, default value none
    • can be used in tag-based JavaScript callbacks to find all matching blog articles by their tag property value
  • category: optional, type string, default value none
    • can be used in tag-based JavaScript callbacks to find all matching blog articles by their category property value

Blog Related Data Catalogs For Implementing Sophisticated Blog Sites

Trio uses the above blog-specific front matter properties to generate numerous blog related data catalogs for you. You can access these data catalogs in your tag-based JavaScript callbacks and you can view them in file root/trio.manifest.json.

There are many ways you can take advantage of these data catalogs, such as using them to create lists of articles grouped by tags in your tag pages or lists of articles grouped by category in your category pages, for instance. These are common elements in many sophisticated blog sites and Trio makes it easy for you to implement them in your site.
